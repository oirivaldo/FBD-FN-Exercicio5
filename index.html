Normalização de Banco de Dados

Forma Não Normalizada Primeira Forma Normal (1FN) Segunda Forma Normal (2FN) Terceira Forma Normal (3FN)

Forma Não Normalizada (UNF)

A tabela contém atributos multivalorados, o que viola a normalização.

Cliente(idCliente, nomeCliente, (idTrans, dataTrans, valorTrans))
Pedido(numPed, codCli, dataPed, nomeCli, codCidCli, nomeCidCli, (codProd, nomeProd, quant, valorUni, valorTotal))

Primeira Forma Normal (1FN)

Removemos atributos multivalorados e criamos tabelas separadas.

Cliente(idCliente, nomeCliente)
Transacao(idTrans, idCliente, dataTrans, valorTrans)
Pedido(numPed, codCli, dataPed)
ItensPedido(numPed, codProd, quant, valorUni, valorTotal)
Produto(codProd, nomeProd)
Cidade(codCidCli, nomeCidCli)

Segunda Forma Normal (2FN)

Eliminamos dependências parciais, organizando melhor os relacionamentos.

Cliente(codCli, nomeCli, codCidCli)
Cidade(codCidCli, nomeCidCli)
Pedido(numPed, codCli, dataPed)
ItensPedido(numPed, codProd, quant, valorUni, valorTotal)
Produto(codProd, nomeProd)
Transacao(idTrans, idCliente, dataTrans, valorTrans)

Terceira Forma Normal (3FN)

Removemos dependências transitivas e eliminamos atributos calculáveis.

Cliente(codCli, nomeCli, codCidCli)
Cidade(codCidCli, nomeCidCli)
Pedido(numPed, codCli, dataPed)
ItensPedido(numPed, codProd, quant, valorUni)
Produto(codProd, nomeProd)
Transacao(idTrans, idCliente, dataTrans, valorTrans)
